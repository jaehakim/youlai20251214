## 프로젝트 소개

[vue3-element-admin](https://gitcode.com/youlai/vue3-element-admin) Vue3, Vite7, TypeScript 및 Element-Plus를 기반으로 구축된 초간단 즉시 사용 가능한 엔터프라이즈급 백오피스 관리 프론트엔드 템플릿입니다. 매칭되는 Java 백엔드 [youlai-boot](https://gitee.com/youlaiorg/youlai-boot) 및 Node 백엔드 [youlai-nest](https://gitee.com/youlaiorg/youlai-nest)가 있습니다. 개발을 간편하게 하기 위해 간소화 버전 [vue3-element-template](https://gitee.com/youlaiorg/vue3-element-template) 및 JavaScript 버전 [vue3-element-admin-js](https://gitee.com/youlaiorg/vue3-element-admin)도 제공합니다.


## 프로젝트 특징

- **간결하고 사용하기 쉬움**: [vue-element-admin](https://gitee.com/panjiachen/vue-element-admin)을 기반으로 업그레이드된 Vue3 버전으로, 과도한 캡슐화 없이 쉽게 시작할 수 있습니다.
- **데이터 상호작용**: `Mock` 데이터 및 [온라인 인터페이스 문서](https://www.apifox.cn/apidoc/shared-195e783f-4d85-4235-a038-eec696de4ea5)를 지원하며, 대응하는 [Java](https://gitee.com/youlaiorg/youlai-boot) 및 [Node](https://gitee.com/youlaiorg/youlai-nest) 백엔드 소스코드를 제공합니다.

- **시스템 기능**: 사용자 관리, 역할 관리, 메뉴 관리, 부서 관리, 사전 관리, 시스템 설정, 공지사항 등의 기능 모듈을 제공합니다.
- **권한 관리:** 동적 라우트, 버튼 권한, 역할 권한 및 데이터 권한 등 다양한 권한 관리 방식을 지원합니다.

- **인프라 기능**: 국제화, 다중 레이아웃, 다크 모드, 전체 화면, 워터마크, 인터페이스 문서 및 코드 생성기 등의 기능을 제공합니다.
- **지속적인 업데이트**: 프로젝트는 지속적으로 오픈소스로 업데이트되며, 도구와 의존성을 실시간으로 업데이트합니다.

## 프로젝트 시작

- **환경 준비**

| 환경 유형 | 버전 요구사항 | 비고 |
|---------|---------|------|
| **Node.js** | `^20.19.0` 또는 `>=22.12.0` | LTS 버전 사용 권장 (주 버전은 짝수) |
| **패키지 매니저** | `pnpm >= 8.0.0` | 프로젝트에서 pnpm을 패키지 매니저로 사용 |
| **개발 도구** | [Visual Studio Code](https://code.visualstudio.com/Download) | Vue 및 TypeScript 관련 플러그인 설치 권장 |


- **빠른 시작**

```bash
# 코드 복제
git clone https://gitee.com/youlaiorg/vue3-element-admin.git

# 디렉토리 전환
cd vue3-element-admin

# pnpm 설치
npm install pnpm -g

# 미러 소스 설정 (선택 사항)
pnpm config set registry https://registry.npmmirror.com

# 종속성 설치
pnpm install

# 실행 시작
pnpm run dev
```


## 프로젝트 배포

`pnpm run build` 명령을 실행한 후, 프로젝트는 패키징되어 `dist` 디렉토리를 생성합니다. 다음으로, `dist` 디렉토리의 파일을 서버의 `/usr/share/nginx/html` 디렉토리로 이동하고, Nginx를 설정하여 리버스 프록시를 구성합니다.

```bash
pnpm run build
```

다음은 Nginx 설정 예시입니다:

```nginx
server {
    listen      80;
    server_name localhost;

    location / {
        root   /usr/share/nginx/html;
        index  index.html index.htm;
    }

    # 리버스 프록시 설정
    location /prod-api/ {
        # api.youlai.tech를 본인의 백엔드 API 주소로 변경하고, 뒤의 슬래시 /를 유지하세요
        proxy_pass http://api.youlai.tech/;
    }
}
```

더 자세한 정보는 이 글을 참조하세요: [Nginx 설치 및 구성](https://blog.csdn.net/u013737132/article/details/145667694).

## 로컬 Mock

프로젝트는 온라인 및 로컬 Mock 인터페이스를 모두 지원합니다. 기본적으로 온라인 인터페이스를 사용하며, Mock 인터페이스로 변경하려면 `.env.development` 파일의 `VITE_MOCK_DEV_SERVER`을 `true`로 수정하면 됩니다.

## 백엔드 인터페이스

> Java 개발 기초가 있다면, 다음 단계를 따라 온라인 인터페이스를 로컬 백엔드 인터페이스로 전환하여 엔터프라이즈급 프론트엔드-백엔드 분리 개발 환경을 구축할 수 있습니다.

1. `Java` 및 `SpringBoot`로 개발된 백엔드 [youlai-boot](https://gitee.com/youlaiorg/youlai-boot.git) 소스코드를 가져옵니다.
2. 백엔드 프로젝트의 설명 문서 [README.md](https://gitee.com/youlaiorg/youlai-boot#%E9%A1%B9%E7%9B%AE%E8%BF%90%E8%A1%8C)에 따라 로컬 시작을 완료합니다.
3. `.env.development` 파일의 `VITE_APP_API_URL` 값을 https://api.youlai.tech에서 http://localhost:8989로 변경합니다.


## 주의사항

- **자동 임포트 플러그인 자동 생성 기본값 비활성화**

  템플릿 프로젝트의 컴포넌트 타입 선언이 이미 자동 생성되어 있습니다. 새 컴포넌트를 추가하고 사용하려면, 그림에 표시된 방법에 따라 자동 생성을 활성화하세요. 자동 생성이 완료된 후, `false`로 설정하여 중복 실행으로 인한 충돌을 피하세요.

  ![](https://foruda.gitee.com/images/1687755823137387608/412ea803_716974.png)

- **프로젝트 시작 후 브라우저 화면이 공백**

  브라우저를 업그레이드하세요. 구 버전의 브라우저는 선택적 연쇄 연산자 `?.`와 같은 새로운 JavaScript 문법을 지원하지 않을 수 있습니다.

- **프로젝트를 저장소의 최신 버전으로 동기화 및 업그레이드 후**

  `pnpm install`로 업데이트된 의존성을 설치한 후 시작하는 것을 권장합니다.

- **프로젝트 컴포넌트, 함수 및 참조가 빨간 색으로 표시**

  VSCode를 재시작해보세요.

- **기타 문제**

  다른 문제나 제안이 있으면 [이슈](https://gitee.com/youlaiorg/vue3-element-admin/issues/new)를 열어주세요.
